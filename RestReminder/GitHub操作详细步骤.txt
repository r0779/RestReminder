=========================================
    GitHub Actions 详细操作步骤
=========================================

步骤 1: 打开 GitHub Actions 页面
=========================================
浏览器会自动打开此页面：
https://github.com/r0779/RestReminder/actions

如果没有自动打开，请手动打开上面的链接


步骤 2: 找到并点击工作流
=========================================

方法 A - 如果有自动构建正在运行：
--------------------------------------
1. 在页面左侧，你会看到一个工作流列表
2. 找到名为 "Build Android APK" 的工作流
3. 点击它

方法 B - 如果需要手动触发构建：
--------------------------------------
1. 在页面左侧的工作流列表中
2. 找到 "Build Android APK"
3. 点击这个工作流名称

工作流名称显示位置：
┌─────────────────────────────────────┐
│  左侧菜单栏                          │
│  ┌───────────────────────────────┐ │
│  │ Workflows                      │ │
│  │   ✓ Build Android APK  ← 点击这里 │ │
│  │   ● other-workflow              │ │
│  └───────────────────────────────┘ │
│                                     │
│  右侧内容区域                        │
└─────────────────────────────────────┘


步骤 3: 点击 Run workflow 按钮
=========================================

进入 "Build Android APK" 工作流页面后：

1. 向下滚动，找到右侧的按钮区域
2. 找到 "Run workflow" 按钮（通常是蓝色按钮）
3. 点击 "Run workflow"

按钮位置示例：
┌─────────────────────────────────────┐
│  Workflows > Build Android APK      │
├─────────────────────────────────────┤
│                                     │
│  [Recent workflow runs]             │
│                                     │
│  ┌───────────────────────────────┐ │
│  │ Run workflow  ← 点击这里        │ │
│  │           （蓝色按钮）           │ │
│  └───────────────────────────────┘ │
│                                     │
└─────────────────────────────────────┘


步骤 4: 选择分支并运行
=========================================

点击 "Run workflow" 后，会弹出一个对话框：

1. 在 "Use workflow from" 下拉框中选择：main
2. 点击绿色的 "Run workflow" 按钮

对话框示例：
┌─────────────────────────────────────┐
│  Run workflow                         │
├─────────────────────────────────────┤
│  Use workflow from:                  │
│  [ main        ▼ ]  ← 选择 main       │
│                                     │
│  [ Run workflow ]  ← 点击绿色按钮    │
│                                     │
└─────────────────────────────────────┘


步骤 5: 等待构建完成
=========================================

1. 页面会自动跳转到构建详情页
2. 等待 3-5 分钟
3. 构建状态会显示：
   - 🟡 In progress（进行中）
   - ✅ Success（成功）
   - ❌ Failed（失败）

构建状态示例：
┌─────────────────────────────────────┐
│  Build Android APK #123              │
├─────────────────────────────────────┤
│  Status: ✅ Success (2 minutes ago)  │
│                                     │
│  [Steps:                            │
│   ✓ Checkout code                   │
│   ✓ Set up JDK 17                   │
│   ✓ Grant execute permission        │
│   ✓ Build with Gradle               │
│   ✓ Upload APK                     │
│  ]                                  │
└─────────────────────────────────────┘


步骤 6: 下载 APK
=========================================

构建成功后：

1. 在构建详情页，向下滚动
2. 找到 "Artifacts" 部分
3. 点击 "rest-reminder-apk" 下载

下载区域示例：
┌─────────────────────────────────────┐
│                                     │
│  Artifacts                           │
│  ┌───────────────────────────────┐ │
│  │ rest-reminder-apk  ← 点击下载  │ │
│  │ Duration: 2 minutes            │ │
│  └───────────────────────────────┘ │
│                                     │
└─────────────────────────────────────┘


步骤 7: 解压并安装
=========================================

1. 下载的文件是 .zip 格式
2. 解压 zip 文件
3. 找到 `app-debug.apk` 文件
4. 传输到手机并安装


=========================================
    常见问题
=========================================

Q: 找不到 "Run workflow" 按钮？
A: 说明代码推送已经触发了自动构建，直接等待构建完成即可

Q: 构建失败怎么办？
A: 查看构建日志，找到错误原因，修改代码后重新推送

Q: 可以直接下载旧版本的 APK 吗？
A: 可以，在 GitHub Actions 页面找到旧的构建记录，下载对应的 Artifacts
