=========================================
    APK 编译方案说明
=========================================

方案 1: GitHub Actions 在线编译（推荐）
=========================================
优点：
  - 不需要本地配置 Gradle
  - 不需要等待本地编译
  - 自动化，可靠
  - 可以构建任意历史版本

步骤：
  1. 双击运行: 推送到GitHub并构建.bat
  2. 自动推送到 GitHub
  3. 在浏览器中点击 "Run workflow"
  4. 等待 3-5 分钟
  5. 下载生成的 APK

时间：5-10 分钟（大部分时间无需等待）


方案 2: 本地命令行编译
=========================================
优点：
  - 完全本地操作
  - 可以立即得到 APK

步骤：
  1. 双击运行: 立即编译.bat
  2. 等待 3-5 分钟
  3. 自动打开 APK 文件夹

时间：3-5 分钟（需要全程等待）


方案 3: Android Studio 图形界面
=========================================
优点：
  - 可视化操作
  - 可以调试和修改代码

步骤：
  1. 打开 Android Studio
  2. File -> Open -> 选择项目文件夹
  3. 等待 Gradle 同步
  4. Build -> Build APK(s)

时间：5-15 分钟（首次同步较慢）


=========================================
    快速推荐
=========================================

如果您现在就想拿到 APK：
  → 使用方案 1（GitHub Actions）

如果您需要经常修改和编译：
  → 使用方案 2（本地命令行）

如果您需要调试代码：
  → 使用方案 3（Android Studio）
